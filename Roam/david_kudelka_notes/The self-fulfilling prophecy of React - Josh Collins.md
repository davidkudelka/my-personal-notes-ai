  * notes:
    * ### React isn't great at anything expect being popular
      * This is bit of an overstatement, but it's true, even tho React inspired many great improvements many frameworks can do the same or do it even better
      * React has aged. And I don't think most people - particularly those using it regularly - realise how much or how poorly
    * ## If you are choosing to do new project there is list of requirements that you should be interested in
      * List
        * Performance
        * Learning Curve
        * Bundle size
        * Scalability
        * Community and support
        * Financial backing
        * Developer experience
        * Hireability
      * ### Performance
        * React is lacking behind all other competitors like Vue, Svelte, Solid, Inferno or Preact
        * [Link to some performance comparison](https://stefankrause.net/js-frameworks-benchmark8/table.html)
      * ### Learning Curve
        * It's harder to learn React than you might think
          * It requires a lot of boilerplating or manual handle to achieve some best practices other frameworks do for you out of the box
          * Memoisation of components
          * `JSX` can be tricky for new-comers 
          * Preventing infinite loops in `useEffect` is your responsibility
      * ### Bundle size
        * React has quite big bundle size, most of the modern browsers do not support shared cache of libraries between domains
      * ### Community and support
        * React has the biggest community, but it doesn't have to be such huge advantage
        * Sometimes big enough is enough and more people doesn't bring any extra value 
      * ### Developer experience
        * React’s satisfaction and interest have been declining steadily for years, while its usage has flatlined
      * ### Hireability
        * React has the biggest advantage
        * But it still doesn't have to be good advatnage - because
          * You will have more complex code, bigger bundle size, less satisfied people just for short-term gains of fast on-boardings
    * ## So why does React stay on top?
      * Because we don't always value the strongest choice as much as we value consensus
      * Companies keep choosing React, because they know there's massive pool of candidates who know it. Candidates keep learning React because they know companies are hiring for it -> It's a self-sustaining cycle
      * ### When and how does this change?
        * It can happen sudden or slowly, but most of the time the change happens slowly, year over year of decline
        * Some companies might already start to choose different option, because they can't afford React's performance
        * It's possible that we are already moving past React, but we can't see it at a high level yet
