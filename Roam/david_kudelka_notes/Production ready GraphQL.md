    * ### Why was something like GraphQL created?
      * Some api's had to support a lot of different clients and it was very hard to maintain them all
      * **One-Size-Fits-All**
        * this is type of API design that is trying to solve too many use cases, usually some over-fetching happens, it's hard to optimise for specific use cases
      * **Facebook creating GraphQL**
        * Then facebook created GraphQL specification, because they were thinking about it more in terms of graph, different entities relate to each other like a graph
        * Mainly to solve problem with hard to maintain API's, they wanted to have something easy to understand and good for solving specific client's issues
