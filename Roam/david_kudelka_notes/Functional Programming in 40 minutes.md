    * ### **What is it ?**
      * We can think about more like a **refactoring** how to do programming
      * OO programming leads to mess if you give it some time
        * The next bug is really harder to fix than it really should be
      * Mathematicians and programmers do very similar things, so when coming up with new style we can get inspiration from mathematics
        * They have very similar problems, abstractions, definitions
        * Is there something we can take from them ?
          * Yes, they have functions, it is relationship between 2 sets
          * Our functions are bit different, because in mathematics they have very predictable output in our code if we are adding files, calling network they can be less predictable
          * We can do something that helps get close to those mathematician functions
          * Pure functions - no side effects, there is input arguments and output based on this output
