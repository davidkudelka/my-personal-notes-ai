  * notes:
    * ### List of responsibilities
      * Github reviews
        * Ensure consistent use and implementation of GraphQL across various projects and teams. Manage schema design, API design, and enforce best practices in GraphQL usage
      * Documentation
        * Create and manage guidelines and documentation for GraphQL usage, schema design, and development within the company
        * Ensure that the documentation and learning resources are up-to-date and accessible.
      * Collaboration
        * Facilitate discussion forums and channels where team members can discuss challenges, solutions, and ideas related to GraphQL - #ama-graphql and #t-graphql-guild
      * Improvements
        * Explore, validate, and introduce new GraphQL tools and technologies
        * Drive long-term technical initiatives
    * ### What GraphQL governors shouldn't do
      * Avoid Direct Involvement in Low-Level Decisions
        * Governors should not immerse themselves in granular decision-making processes or individual project management
      * Resist Overseeing Every Detail
        * They should avoid getting into every minutiae of code reviews, schema designs, and other detailed tasks that can be handled by respective team members
      * Stay Away from Direct Maintenance
        * Not getting involved in day-to-day bug fixes, code deployments, and maintenance works
